클라이언트가 계약서 쓰기를 꺼려하면 어떻게 해야 하나요?

소규모 클라이언트들이 "이 정도면 그냥 믿고 하면 되지 않나요?"라고 하는데, 어떻게 설득하거나 대응하는 게 좋을까요?

저는 계약서 없이 진행했다가 범위가 계속 늘어나거나, 수정 요청이 끝없이 와서 힘들었던 경험이 있어서 꼭 써야 한다고 생각하는데... 막상 말꺼내기가 어렵더라고요. 자연스럽게 계약서를 도입하는 방법이 있을까요?

프리랜서 9년 동안 이 문제를 정말 많이 겪었어요. 결론부터 말씀드리면 — 계약서를 꺼리는 클라이언트가 바로 계약서가 가장 필요한 클라이언트입니다.

가장 효과적인 방법은 "계약서 부탁드려요"가 아니라 "작업 범위 확인서를 공유드릴게요"라고 프레이밍을 바꾸는 거예요. 심리적 저항이 훨씬 줄어들어요.

실전 스크립트

"원활한 작업 진행을 위해 작업 범위와 일정을 문서로 정리해서 공유드릴게요. 나중에 서로 오해 없이 진행하기 위해서예요." — 이렇게 말하면 대부분 자연스럽게 받아들여요.

계약서라는 말 대신 쓸 수 있는 표현들: 작업 범위 확인서, 프로젝트 요약서, 업무 협약서, 견적 확인서. 내용은 동일해도 이름만 바꾸면 훨씬 자연스럽게 넘어가요.

최소한 포함해야 할 항목들
✓ 작업 범위 (무엇을 만드는지)
✓ 수정 횟수 제한 (몇 회까지 무료인지)
✓ 납기일 / 결과물 전달 형식
✓ 금액과 지급 일정 (선금 30~50% 추천)
✓ 저작권 귀속 조건
✓ 추가 작업 발생 시 단가
⚠️ 중요: 선금 없이 작업을 시작하지 마세요. "믿고 하면 된다"는 말 뒤에 돈 못 받는 경우가 많아요. 선금 30~50%는 진심으로 하는 클라이언트라면 다 내줍니다.

만약 그래도 계약서를 완강히 거부한다면, 그 클라이언트와의 작업을 재고해보세요. 신뢰는 계약서와 함께 가는 거지, 계약서 대신 신뢰를 믿는 건 아니에요.

저도 비슷한 경험이 있어요. 제가 쓰는 방법은 견적서 자체에 조건을 포함시키는 것이에요.

견적서를 보낼 때 하단에 "아래 조건 확인 후 착수금 입금 시 작업이 시작됩니다"라고 명시하면, 별도로 계약서 얘기를 꺼내지 않아도 자연스럽게 합의가 이뤄져요. 착수금 입금 = 조건 동의가 되는 구조예요.

효과적인 말 하나 더 보태자면 — "저도 선생님을 믿어요. 이건 서로를 위한 거예요. 나중에 기억이 다를 때 서로 답답하지 않으려고요."

계약서를 불신의 상징이 아니라, 서로의 기억을 보호하는 도구로 프레이밍하면 저항이 많이 줄어들어요.
